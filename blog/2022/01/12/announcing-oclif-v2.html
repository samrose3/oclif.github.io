Announcing oclif v2!

January 12, 2022

Hello and happy new year! It's our great pleasure to announce that we have released oclif v2, which uses the new @oclif/core library! ðŸŽ‰

oclif v2 includes many changes that improve the experience of both creating and using an oclif CLI.

Going forward, we don't plan to make any updates to oclif v1 and its corresponding libraries, except for critical security fixes. See the compatibility matrix for a list of these libraries. In order to give developers time to migrate from v1 to v2, we're not completely dropping support yet. But at some point in the future we'll archive the v1 repositories and deprecate their versions on NPM.

Whatâ€™s changing?

Repository Changes

All repositories under the oclif org now use main as their primary branch and oclif-v1 as the legacy branch. The main branch corresponds to oclif v2 and the oclif-v1 branch to oclif v1. If you find any repos that donâ€™t have a main branch, you can safely assume that they're not used in oclif v2 and will be deprecated in the future.

Consolidating Tools & Libraries

The oclif-dev CLI has now been incorporated into oclif. You now no longer need to install a separate package to manage your entire CLIâ€™s lifecycle.

The following libraries have been consolidated under @oclif/core and will be deprecated at some point in the future. Read the migration guide to learn how to update your CLI or plugin to use the new core library.

  • @oclif/command
  • @oclif/config
  • @oclif/error
  • @oclif/help
  • @oclif/parser

No more single vs multi command CLIs

There's only one "type" of oclif v2 CLI, but it can have as few or as many commands as a developer wants. As a result, thereâ€™s only one command to generate CLIs, oclif generate, as opposed to the old oclif single and oclif multi commands.

Node > 12

To ensure oclif CLIs are as secure as possible, all v2 libraries and plugins support only Node 12 or higher as of now. Going forward, they'll support only Node Maintenance or higher, as defined by Node's release schedule here: https://nodejs.org/en/about/releases/.

Whatâ€™s new?

New example CLI

oclif-hello-world is our new example repo. This is also the repo that's used as a template when running oclif generate to create a new CLI. This repo will always be able to be referenced as an example of what an oclif v2 CLI should look like.

Spaced commands

oclif CLIs can now use spaces, instead of colons, to separate topics and subcommands. To enable this feature, simply add â€œtopicSeparator": " " to the oclif config in your package.json. See an example in our example repo.

// Old commands that use :
$ mycli do:something

// New spaced commands
$ mycli do something

Note: Spaced commands are backwards compatible. So if you enable spaced commands for your CLI, users will still be able to use : as a separator. This ensures that any existing scripts don't break.

New Help Output

Weâ€™ve revamped the way command help is outputted to the terminal, making it both easier to implement and easier to read. See the difference between the old help output on the left and the new help output on the right in the screenshots below.

Notice that there are new sections for flags and global flags, examples are displayed with better spacing, and there is a section at the bottom called Configuration Variables. This Configuration Variables section is not part of the new help by default. But we've added support for custom help sections, which is what the sf CLI uses to create the new section.

Async Command Parsing

Weâ€™ve also improved the performance of new CLIs by rewriting how commands are parsed on initialization. The new parser is asynchronous, which makes CLIs with a lot of commands or installed plugins much faster.

Part of our charter for the release of oclif v2 includes improving our engagement with the oclif community. We know that over the past couple of years weâ€™ve reduced our involvement, and a lot of issues and PRs have languished in limbo. Hopefully youâ€™ve already seen increased activity and responses from oclif maintainers recently, but if you havenâ€™t, please donâ€™t hesitate to ping us by tagging @admins in oclif repos.
